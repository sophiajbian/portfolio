Northwestern Formula SAE:
Data Acquisition

Building the systems that collect, display, and interpret real-time vehicle data for Northwestern's Formula SAE race car — from dashboard firmware to custom PCB design.

C++ PCB Design Altium Designer Embedded Systems Git

Overview

The Project

Northwestern Formula Racing's Data Acquisition subteam is responsible for collecting and visualizing real-time telemetry from the race car during testing and competition. As a member of the DAQ subteam, I've worked on both the software and hardware sides — contributing to the dashboard display system and designing a revised PCB for the dashboard controller.

The dashboard system gives the driver and engineers immediate feedback on vehicle state, including speed, temperatures, and system health. Visual indicators like NeoPixel LEDs provide at-a-glance status information even in bright outdoor conditions.

Hardware

The PCB

I designed Version 2 of the dashboard controller PCB in Altium Designer, improving on the previous iteration with better component placement and layout for the car's packaging constraints.

Software

The Firmware

The dashboard firmware is written in C++ and responsible for parsing incoming CAN messages, updating the display, and driving the LED indicators based on vehicle state.

  • Developed features for a C++-based dashboard system to display real-time vehicle telemetry
  • Designed and tested NeoPixel LED drivers for visual status indicators on the car
  • Collaborated across subteams using Git-based workflows and code review
  • Participated in testing sessions to validate behavior under real operating conditions
